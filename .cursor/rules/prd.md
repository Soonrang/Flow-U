# 📄 FlowU PRD (Product Requirements Document)

## 1. 프로젝트 개요 (Project Overview)
* **제품명:** FlowU (플로우유)
* **한 줄 소개:** 유기동물 보호소의 위치와 정보를 직관적으로 제공하고 봉사 및 입양 참여를 독려하는 지도 기반 플랫폼.
* **해결하고자 하는 문제:** * 기존 유기동물 보호소 정보가 파편화되어 있어 일반인이 접근하기 어려움.
  * 시각적으로 딱딱하고 복잡한 공공데이터 대신, 친근하고 따뜻한 UX/UI가 필요함.
* **타겟 사용자:** 유기동물 봉사활동이나 입양에 관심 있는 일반인, 보호소를 운영/홍보하고자 하는 관리자.

## 2. 핵심 가치 및 디자인 원칙 (Core Values & Design Principles)
* **친근함 (Friendly):** 파스텔 톤(핑크, 블루, 에메랄드)과 둥근 UI(`rounded-2xl`)를 사용하여 심리적 진입 장벽을 낮춤.
* **직관성 (Intuitive):** 카카오맵 기반의 시각적 마커(🐱, 🐶, 🐾)를 통해 보호소의 주 보호 동물 유형을 한눈에 파악.
* **신뢰성 (Reliable):** 사용자가 직접 보호소를 등록하되, 관리자 승인(`aprv_status`)을 거친 검증된 데이터만 지도에 노출.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 인증 및 진입 (Auth & Entry)
* **메인 랜딩 화면:** FlowU 로고 및 서비스 소개 노출.
* **로그인:** 카카오 소셜 로그인을 통한 간편 인증 지원.
* **게스트 모드:** '로그인 없이 이용하기' 버튼을 통해 지도 페이지로 즉시 진입 가능.

### 3.2. 지도 및 마커 (Map & Markers)
* **지도 엔진:** 카카오맵 API (`react-kakao-maps-sdk`) 활용.
* **커스텀 마커:** * 고양이(Cat): 핑크색 테마 + 🐱 아이콘
  * 강아지(Dog): 파란색 테마 + 🐶 아이콘
  * 혼합(Mixed): 에메랄드 테마 + 🐾 아이콘
* **인포윈도우:** 마커 호버 또는 클릭 시 보호소 이름과 간략한 정보 말풍선 노출.
* **데이터 필터링:** DB에서 `use_yn = 'Y'` 및 `aprv_status = 'Y'`(승인 완료)인 데이터만 지도에 렌더링.

### 3.3. 검색 및 필터링 (Search & Filter)
* **검색 오버레이:** 지도 상단에 둥근 형태의 검색창 오버레이 배치.
* **지역 기반 필터링:** `sido`(광역 지자체)와 `sigungu`(기초 지자체) 컬럼을 활용하여 빠르고 정확한 지역 검색 지원.

### 3.4. 보호소 등록 (Shelter Registration)
* **플로팅 액션 버튼 (FAB):** 우측 하단 `(+)` 버튼을 통해 등록 모달 호출.
* **등록 폼 (Modal):** * 입력 항목: 보호소 이름, 연락처, 주 보호 동물(cat/dog/mixed), 상세 설명.
  * **주소 검색:** '다음 우편번호 API'를 연동하여 주소 입력 시 `sido`, `sigungu`, `address`, `latitude`, `longitude`를 자동 추출 및 저장.
* **상태 처리:** 등록 완료 시 DB에 `aprv_status = 'W'`(대기) 상태로 저장되어 관리자 승인 전까지 노출 제한.

### 3.5. 관리자 워크플로우 (Admin Workflow)
* **권한 제어:** Supabase RLS 및 관리자 권한 검증을 통해 보호된 페이지.
* **승인/거절:** 대기(`W`) 상태의 신규 보호소 내역을 확인하고 승인(`Y`) 또는 거절(`R`) 처리.
* **논리적 삭제:** 운영이 중단된 보호소는 `use_yn = 'N'`으로 업데이트하여 지도에서 숨김.

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1. 기술 스택
* **Frontend:** Next.js (App Router), TypeScript, Tailwind CSS, Lucide React
* **Backend/DB:** Supabase (PostgreSQL, Auth)
* **Infra:** Vercel (CI/CD 자동화 배포)

### 4.2. 성능 및 보안
* **보안 (Security):** Supabase RLS(Row Level Security)를 활성화하여 익명 키(`Anon Key`) 탈취 시 데이터 임의 조작 방지. 환경변수(`.env.local`) 엄격 관리.
* **반응형 (Responsive):** 모바일, 태블릿, 데스크톱 환경을 모두 고려한 반응형 UI 오버레이 구성.

## 5. 향후 확장 계획 (Future Scope - Phase 2)
* **커뮤니티/공지사항:** 각 보호소 마커 클릭 시 진입하는 상세 페이지에 게시글(`posts`) 및 댓글(`comments`) 기능 추가.
* **봉사/입양 신청:** 보호소 상세 페이지에서 직접 봉사 일정 예약 또는 입양 문의를 남기는 기능 도입.
